# Citation Foundation Status 4H

## Purpose

This checkpoint records the completed Sprint 4H citation foundation after:

- `3fb5ba5 feat: add human apa verification gate mvp`

The 4H series moved ATP Knowledge Studio from compact SourceCard citation notes toward a durable, review-gated citation foundation. It did not create APA-final references, automatic metadata extraction, DOI lookup, web search, AI/API integration, or downstream DraftArtifact/export consumption.

## Completed 4H Flow

Completed citation foundation:

- Human SourceCard metadata completion.
- Structured bibliographic metadata contract.
- Structured metadata persistence.
- Structured metadata readiness validator.
- APA Reference Candidate contract.
- APA Reference Candidate Preview.
- Human APA Verification Gate Plan.
- Human APA Verification Gate MVP.

Current flow:

```text
Saved SourceCard
-> human compact metadata completion
-> structured bibliographic metadata entry
-> structured metadata read-back
-> structured metadata readiness validation
-> APA Reference Candidate Preview
-> Human APA Verification Gate
-> saved APA reference review artifact
-> future downstream citation readiness consumption
```

## What Is Persistent

Persistent layers now include:

- Compact SourceCard metadata updates through `update_source_card_metadata`.
- Structured bibliographic metadata in `source_card_bibliographic_metadata`.
- Human APA reference review artifacts in `source_card_apa_reference_reviews`.

The APA review artifact stores:

- candidate reference text
- verified reference text
- verification status
- verification scope
- checklist snapshot
- reviewer note
- source metadata snapshot
- accepted warnings
- resolved blockers
- APA style version
- human review timestamp

## What Remains Preview-Only

Preview-only layers:

- Structured metadata readiness mapper.
- APA Reference Candidate Preview.
- Candidate reference text generated by the deterministic local preview mapper.

These previews do not mutate SourceCards, structured metadata, DraftArtifacts, export packages, parser traces, or AI/provider state.

## What Remains Internal-Use Only

The Human APA Verification Gate MVP supports:

- `needs_correction`
- `verified_for_internal_use`

`verified_for_internal_use` means the reviewer accepted the reference for private drafting, internal notes, teaching preparation, or non-publication workflow context. It is not publication-ready and must not be treated as APA-final.

## What Is Still Not APA-Final

Not APA-final:

- compact SourceCard `citationReadiness: ready`
- structured metadata `apaReadiness`
- APA Reference Candidate Preview
- `verified_for_internal_use`
- DOCX MVP export output
- DraftArtifact citation readiness

`apa_final_verified` remains future work. No automated process, deterministic mapper, parser output, AI response, DOI lookup, or external metadata provider may set it.

## Why SourceCard CitationText Is Not Overwritten

`source_cards.citation_text` remains a compact backward-compatible field. It may contain an interim human note or display string, but it is not a structured APA artifact.

The APA review gate does not overwrite `citation_text` because:

- it would blur compact metadata with APA review artifacts
- downstream features may still rely on the historical SourceCard row
- review artifacts need their own snapshot, checklist, scope, and timestamp
- internal-use verification must remain distinct from publication-level APA verification

## Why Verified APA Artifacts Are Separate

Verified APA artifacts are stored separately so ATP can preserve:

- what candidate was reviewed
- what metadata snapshot was used
- what warnings were accepted
- what blockers were resolved
- who/when review happened later when reviewer identity exists
- whether the result is internal-use only or future APA-final

This separation supports auditability and future stale-review detection when SourceCard metadata changes.

## Remaining Limitations

Remaining limitations:

- No APA-final verification workflow.
- No citation manager integration.
- No DOI, ISBN, Crossref, or OpenAlex lookup.
- No automatic metadata extraction.
- No external metadata confidence scoring.
- No stale-review invalidation after metadata edits.
- No normalized checklist history table.
- No downstream DraftArtifact/export consumption of saved APA review artifacts.
- Source Library remains dense and needs progressive workflow redesign.

## Next Direction

The next major roadmap direction should reduce manual citation workload without sacrificing reliability:

- Batch Research Intake Queue.
- Multi-file PDF/DOCX import.
- External metadata cross-check.
- Crossref / OpenAlex / DOI / ISBN metadata matching.
- Confidence scoring.
- Suggested metadata corrections.
- Pending review queues.
- Batch approve / reject / edit.
- Human-verified audit trail.

ATP must not become mostly manual. Automation should produce reviewable suggestions, not unreviewed truth.
