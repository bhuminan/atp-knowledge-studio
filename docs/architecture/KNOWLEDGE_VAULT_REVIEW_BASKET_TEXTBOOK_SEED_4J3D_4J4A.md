# Knowledge Vault Review Basket And Textbook Seed 4J-3D / 4J-4A

## A. Purpose

4J-3D / 4J-4A adds two preview-only bridge layers to the Source Library parsed-DOCX workflow:

- Knowledge Vault Review Basket
- Textbook Request Seed Preview

The goal is to make the workflow feel closer to:

```text
Input -> Automatic Classification -> Keep Records by Tag -> Textbook Request -> DOCX Output
```

This sprint does not save Knowledge Vault records, generate final textbook prose, create DraftArtifacts, or export DOCX.

## B. Relationship To 4J-3A And 4J-3B/4J-3C

4J-3A added deterministic parsed-DOCX classification and tag preview.

4J-3B / 4J-3C mapped classification output into preview-only Knowledge Vault candidate records.

4J-3D / 4J-4A builds on those candidate records by:

- creating a deterministic recommended review basket
- creating a textbook request seed from the review basket and classification textbook relevance

The new previews use frontend state only and remain unsaved.

## C. Review Basket Rules

The review basket is generated by `ParsedDocxKnowledgeVaultReviewBasketMapper`.

Inputs:

- Knowledge Vault candidate preview status
- candidate tag labels
- candidate confidence
- suggested vault use

The mapper returns:

- status
- recommended review items
- basket summary counts
- blockers
- warnings

High-confidence concept or textbook-section candidates are prioritized for review. Medium-confidence candidates are included with human-review-required language. Low-confidence candidates stay lower priority.

No candidate is auto-approved, saved, verified, or written to the Knowledge Vault.

## D. Textbook Request Seed Behavior

The textbook request seed is generated by `ParsedDocxTextbookRequestSeedMapper`.

Inputs:

- review basket
- classification textbook relevance
- supporting tag labels
- supporting candidate IDs

The mapper returns:

- suggested textbook topics
- possible chapter use
- supporting tags and candidate IDs
- evidence readiness
- request seed title, audience, output type, and readiness
- missing evidence warnings
- guardrail warnings

The preview can suggest a section brief or request seed direction, but it does not generate final chapter prose, call AI, create DraftArtifact records, or export DOCX.

- API keys
- AI/API behavior
- PDF/OCR implementation
- dependencies
- existing save/review gates

## G. Guardrails

Visible guardrails include:

- preview only
- no automatic save
- human review required
- no AI used
- no citation finality
- SourceCard `citationText` not overwritten
- APA-final not supported
- DraftArtifact remains mock/not-final
- DOCX export remains gated

## I. Remaining Limitations

- Review basket state is deterministic and not user-persisted.
- Textbook request seed is not a generated textbook request record.
- Evidence readiness is a local preview hint only.
- Missing evidence warnings are conservative and not exhaustive.
- Later DraftArtifact and DOCX export paths remain gated.
