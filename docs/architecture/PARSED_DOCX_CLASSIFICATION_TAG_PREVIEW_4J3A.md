# Parsed DOCX Classification Tag Preview 4J-3A

## A. Purpose

4J-3A adds a preview-only automatic classification and tag suggestion layer to the Source Library parsed-DOCX workflow.

The goal is to make the workflow feel more functional after input while preserving conservative source, metadata, citation, and persistence boundaries.

Target workflow remains:

```text
Input -> Automatic Classification -> Keep Records by Tag -> Textbook Request -> DOCX Output
```

This sprint covers the Automatic Classification preview only. It does not create final metadata, final tags, textbook prose, or saved classification records.

## B. Relationship To 4J-1A Through 4J-2C

4J-1A defined the one-screen Source Library information architecture.

4J-1B implemented the workflow shell.

4J-1C clarified the parsed-DOCX intake path and real-vs-mock empty state.

4J-1D / 4J-2A added the guided action path and demoted the secondary workbench.

4J-2B / 4J-2C connected guided action controls to existing UI sections and clarified real parsed-DOCX context.

4J-3A adds the first deterministic classification preview inside that existing shell.

## C. Classification Preview Rules

The preview is generated by `ParsedDocxClassificationPreviewMapper`.

Inputs are existing frontend state only:

- selected local file metadata
- parsed DOCX extraction response
- extracted cleaned/raw text
- extracted segments and segment tags
- parser confidence metadata

The mapper uses deterministic keyword and controlled taxonomy matching only. It does not call a backend command, AI model, provider API, network service, citation service, or metadata lookup.

Suggested source type is conservative:

- `article`
- `book_chapter`
- `report`
- `teaching_note`
- `unknown`

When no conservative rule matches, the preview returns `unknown` with human review required.

## D. Tag Suggestion Behavior

Suggested marketing tags are derived from local controlled taxonomy terms and aliases.

Each suggested tag shows:

- label
- category
- confidence: `high`, `medium`, or `low`
- reason

Suggested tags are preview-only. They are not saved, approved, attached to a SourceCard, or converted into MarketingTag records automatically.

## E. Textbook Relevance Behavior

Textbook relevance is mapped from matched marketing taxonomy categories to possible textbook sections, such as:

- service quality and service experience
- consumer behavior and decision journeys
- marketing strategy and positioning
- brand strategy and brand equity
- digital marketing and analytics

If no strong marketing concept is detected, the preview shows `Needs human textbook mapping`.

This sprint does not generate textbook prose or DraftArtifact content.

## H. Guardrails

Visible compact guardrails include:

- preview only
- no AI used
- no automatic save
- human review required
- SourceCard `citationText` not overwritten
- APA-final not supported
- DraftArtifact remains mock/not-final
- export remains gated

The preview also states that suggested tags are not saved records and that no SourceDocument, SourceCard, MarketingTag, or KnowledgeCard is persisted by the classifier.

## J. Remaining Limitations

- The classifier is deterministic and conservative; it is not a semantic AI classifier.
- Confidence is a local UI hint, not an academic quality score.
- Suggested tags are not saved or linked until existing explicit save/review flows are used.
- Source type remains a hint and does not mutate SourceCard metadata.
- Textbook relevance is a section hint only, not generated prose.

## K. Recommended Next Sprint

Recommended next sprint: 4J-3B reviewed classification state.

Focus on a local human review layer for accepting, rejecting, or deferring classification suggestions before connecting reviewed suggestions to existing explicit MarketingTag and KnowledgeCard save paths. Keep persistence explicit and avoid automatic metadata mutation.
