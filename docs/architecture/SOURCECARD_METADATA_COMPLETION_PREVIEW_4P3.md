# SourceCard Metadata Completion Preview 4P-3

## Purpose

Sprint 4P-3 adds a read-only SourceCard Metadata Completion Preview for a selected saved SourceDocument.

This sprint is a preview/design layer only. It does not save metadata, create SourceCards, add active metadata editing, call backend write commands, change schema, add migrations, modify parser/classification/AI/provider behavior, change citation/APA/export/Writer behavior, add dependencies, or change lockfiles.

## What Completion Preview Was Added

The selected saved SourceDocument detail now shows a compact section labeled:

```text
SourceCard Metadata Completion Preview
```

The section states:

```text
Preview only — metadata is not saved and no SourceCard is created.
```

The preview appears near the SourceCard Metadata Review Gate Preview and before the compact audit trace. It uses a pure frontend mapper and existing saved SourceDocument root fields only.

## Field Groups And Statuses

The preview groups future metadata review fields into five categories.

### Root Identity

Fields:

- SourceDocument title
- file name
- source type/file type
- intake candidate id/provenance

These fields use saved SourceDocument root data. Present values are marked `available`. Missing identity fields show `Not provided yet` and are not guessed.

### Bibliographic Essentials

Fields:

- authors
- year
- source type
- title confirmation

Authors and year are not stored on the SourceDocument root, so they show `Needs review`. Source type and title confirmation use existing root values as preview context, but future review must still confirm their bibliographic meaning.

### Source-Type-Specific Fields

Fields:

- journal/container
- publisher
- DOI
- URL
- volume/issue/pages if applicable

These values are not inferred. They show `Needs review` until a future human metadata completion workflow exists.

### Citation Readiness

Fields:

- citation text
- APA candidate
- human review

Citation text is not verified or generated by this preview. APA candidate state remains non-final. Human review remains required before any future SourceCard creation path.

### Future Approval

Fields:

- explicit metadata review
- explicit SourceCard creation approval

Both remain required future steps. The preview does not satisfy them.

Field statuses:

- `available`
- `needs_review`
- `not_applicable`
- `blocked`

## No-Fabrication Rules

The preview uses only:

- saved SourceDocument title
- saved file name
- saved file type/source type
- saved intake candidate id/provenance when present

The preview does not invent:

- authors
- year/date
- DOI
- URL
- journal
- publisher
- container title
- volume
- issue
- page range
- citation text
- APA reference
- APA-final verification

Missing values display as `Needs review` or `Not provided yet`.

## Disabled Future-Only Actions

The preview may display disabled future-only actions:

- `Future: Save reviewed metadata`
- `Future: Create SourceCard after review`

Both are disabled. No active metadata save button and no active SourceCard creation button were added.

## Why No Metadata Is Saved

This sprint is a form-design preview only. A future metadata completion workflow needs a separate approval contract, audit model, read-back receipt, and persistence boundary before any metadata can be saved.

Saving metadata now would blur the SourceDocument-only boundary and could make unreviewed citation fields appear authoritative.

## Why SourceCard Remains Deferred

The SourceDocument root proves local intake identity and read-back visibility only. It does not prove complete bibliographic metadata or citation readiness.

SourceCard creation still requires:

- reviewed bibliographic metadata
- explicit future metadata approval
- explicit SourceCard creation approval
- audit behavior
- read-back verification

## Why Citation And APA Readiness Are Not Implied

Citation and APA metadata require human academic review. The completion preview does not generate citation text, APA candidate references, or APA-final verification.

The preview avoids citation-ready and APA-final wording and keeps SourceCard creation deferred.

## What Remains Not Implemented

- No metadata save.
- No active metadata editing.
- No SourceCard creation.
- No SourceCard metadata approval DTO.
- No backend write command.
- No schema or migrations.
- No SourceCard creation audit event.
- No parser/classification/AI/provider workflow.
- No CitationGuard, APA verification, evidence review, DOCX export, Writer, or network workflow.
- No downstream records.
