# ATP External Audit Questions 4P-11X

These questions are for an external design consultant auditing ATP Knowledge
Studio's current app state.

## Information Architecture

- Is the 4-room frontstage model appropriate: Library, Cabinet, Writer, Art?
- Should the left menu be simplified or collapsed?
- Which current menu items should move to Advanced/Admin?
- Should Dashboard room cards replace most left menu navigation?
- Which routes should remain visible for daily work?
- Which routes are internal modules rather than user-facing rooms?

## Source Library

- What should be the main user task on the Library screen?
- Should Source Library be designed as a review desk rather than a backend
  console?
- What should be hidden behind Inspector/Advanced?
- Which trust details are essential frontstage?
- Which audit/status details are internal/debug?
- How should saved SourceDocument read-back be shown without overwhelming the
  main action?

## Writer

- What should be the main user task on the Writer screen?
- Should Writer start with one selected writing request instead of a broad
  dashboard of validation/provider panels?
- How should citation guard and structure validation be shown without obscuring
  drafting?

## Trust And Safety

- How should backend trust/audit be shown without overwhelming users?
- Should metadata editing remain blocked until IA is simplified?
- What is the safest next sprint?
- Which information is essential for trust, and which is internal/debug?
- What should be removed from frontstage entirely?

## Visual System

- Does the 8-bit/pixel studio identity support the academic workflow?
- Where does pixel styling help wayfinding?
- Where does it add noise?
- Should status colors be simplified into a smaller trust system?
- How can panels be reduced without losing the app's distinctive feel?
